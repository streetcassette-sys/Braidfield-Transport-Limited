Status Engine — Automated State Machine
Initial State
Client submits documents via Client Portal. 12h SLA clock starts automatically.
→
UNDER CLEARING
Set when documents are received and SCO begins processing. Client sees "Under Clearing" in their portal.
Trigger: Document submission confirmed
→
TRANSITING
Set the moment Border Agent uploads the finalized T1. Client is instantly notified. Internal invoices sent to operational roles.
Trigger: Finalized T1 uploaded by Border Agent
→
COMPLETED
Congo T1 uploaded. Transit leg closed. Offloading verified by SCO. Supplier notified — truck empty alert sent.
Trigger: Congo T1 + Offloading verification
SLA Rules — 36-Hour Maximum Intake-to-Transit Window
0h
Start

Client Submits Documents

12-hour SLA clock starts automatically. All staff notified.

12h
Warning

Reminder Email Sent

Automated email to Senior Customs Officer, Territory Manager and Executive if Release Order is missing.

24h
Demand

Demand Email Sent

Escalation — demand email sent to the same 3 parties. Dashboard alert activates.

36h
Breach!

SLA Breached — Hard Limit

Total intake-to-transit time has exceeded the 36-hour maximum. Dashboard flags as critical breach. Requires immediate escalation.

Compliance Monitoring — Per Truck, Per Driver
Fitness Due Date
Captured at onboarding per driver. System monitors daily. Alerts triggered to client at 30 days, 7 days, and on expiry.
Road Tax Due Date
Captured per truck at onboarding. System monitors daily. Alerts sent to client when approaching or expired.
Identity Due Date
Driver identity document expiry tracked. Client alerted via WhatsApp or email. Visible in compliance tab.
